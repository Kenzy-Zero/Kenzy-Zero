### 👋 About

I build **open-source tools that make working with big data simpler** — the kind of
software you `pip install` and it just gets out of your way. My focus is the awkward
middle ground between "pandas crashed" and "do I really need a cluster?": files that
are too big for memory but too small to justify Spark.

- 🦆 **Currently building** [`kenze`](https://github.com/Kenzy-Zero/kenze) — big-file data prep that never runs out of memory
- 🧰 I like **lean tools, one command, no ceremony**
- 🌱 Always shipping — feedback and pull requests welcome

---

### 🦆 Featured — [kenze](https://github.com/Kenzy-Zero/kenze)

> **Big-file data prep that never runs out of memory — an interactive shell *and* a one-line CLI.**
> Clean and reshape CSV / Parquet / JSON / Excel files that are too big for pandas.
> DuckDB does the heavy lifting; kenze makes it effortless and auto-sizes memory so
> your job doesn't crash. No SQL required — but no lock-in either.

```bash
pip install kenze          # then just run:  kenze
```
